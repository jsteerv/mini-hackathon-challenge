Your Best Friend: git status

This command shows you what's happening right now. Use it constantly! It tells you:

  • Which branch you're on
  • Which files have changes (red = not staged)
  • Which files are ready to commit (green = staged)
  • Whether you're ahead or behind the remote

Pro tip: Run git status before and after every command while learning!

The Basic Git Workflow

This is the sequence you'll use dozens of times per day. Let's break it down step by step.

Starting a Repository: git init

git init

This creates a new Git repository in your current folder. Run this once per project in the root directory. It creates a hidden .git folder that stores all your history.

Staging Changes: git add

git add filename.txt
git add .

This moves changes from your working directory to the staging area. The dot (.) means "add everything in this directory."

Saving a Snapshot: git commit

git commit -m "Add user login feature"

This creates a permanent snapshot of everything in the staging area. The -m flag lets you add a message describing what changed and why.

Viewing History: git log

git log --oneline

This shows your commit history. Each commit has a unique ID (hash), author, date, and message.

Press 'q' to quit the log view. Use --graph --all for a visual tree!

Undoing Changes: Your Safety Net

‚Ü©Ô∏è

Made a mistake? No problem! Git makes it easy to undo changes at any stage.

Three Ways to Undo

1. Undo changes in working directory (not staged):
git restore filename.txt
2. Unstage files (remove from staging area):
git restore --staged filename.txt
3. Undo last commit (keep changes in working directory):
git reset HEAD~1

Git gives you multiple safety nets - mistakes are never permanent!

Working with Branches

Create a new branch:
git branch feature-login
Switch to that branch:
git switch feature-login
Create and switch in one command:
git switch -c feature-login
See all branches:
git branch

Connecting to Remote Repositories

This is how you collaborate with others and backup your code to the cloud.

Push sends your commits up. Pull downloads and merges changes from others.

Joining an Existing Project: git clone

git clone https://github.com/username/project.git

This downloads a complete copy of a remote repository - all files, all history, all branches. It automatically sets up the connection to the remote.

Syncing with the Team

git push origin main

Uploads your local commits to the remote repository.

git pull

Downloads changes from the remote and merges them into your current branch. Always pull before starting work to get the latest changes!

Remember: You must commit locally before you can push!
